# GCC Universal Setup – Design Document

**Date:** 2026-02-28
**Status:** Approved
**Goal:** Replace the manual multi-step setup with a single `setup.sh` script that works for any LLM agent, any project state (new or existing), and any OS (Linux/macOS/Windows via Git Bash).

---

## Problem Statement

The current setup requires users to:
1. Manually run `git init`
2. Manually copy the `scripts/` folder
3. Manually run `gcc-init.sh`
4. Manually copy `CLAUDE.md`

This is four steps, Claude-specific, and intimidating for non-technical users. GCC should work with one command for all agents.

---

## Design Decisions

### D1: Single entry point — `setup.sh`

One script in the repo root handles everything. Users run:

```bash
# With arguments (non-interactive):
bash GCC/setup.sh /path/to/project "Project Name" "Project Goal"

# Without arguments (interactive wizard):
bash GCC/setup.sh
```

The script detects whether git is already initialized and skips `git init` if so. It also detects whether an initial commit exists and creates one only if needed.

### D2: `AGENTS.md` as the primary instruction file

`CLAUDE.md` is renamed/replaced by `AGENTS.md` as the human-readable source of truth. The content is rewritten to be agent-agnostic (no "You are Claude" wording).

`AGENTS.md` is recognized natively by:
- OpenAI Codex / ChatGPT agents
- Google Antigravity (also reads `.agent/rules/`)
- Most open-source agent frameworks

### D3: Agent-specific copies generated automatically

`setup.sh` generates copies of `AGENTS.md` content into the exact locations each major agent expects. All copies are identical in content. There is no symlinking (symlinks do not work reliably on Windows/Git Bash).

| Agent | File path |
|---|---|
| OpenAI Codex | `AGENTS.md` ← primary |
| Claude (Anthropic) | `CLAUDE.md` |
| Cursor | `.cursor/rules` |
| Windsurf | `.windsurfrules` |
| GitHub Copilot | `.github/copilot-instructions.md` |
| Gemini CLI | `.gemini/GEMINI.md` |
| Google Antigravity | `.agent/rules/gcc.md` |

### D4: Interactive wizard as fallback

When called without arguments, `setup.sh` prompts the user with clear questions and sane defaults. Output is a clean progress display confirming each step.

### D5: README updated

`README.md` is rewritten to reflect the new one-command setup. The six-step quickstart becomes a two-step quickstart:
1. Clone GCC
2. Run `setup.sh`

Agent compatibility table is added to README.

### D6: Existing `gcc-init.sh` stays unchanged

`setup.sh` calls `gcc-init.sh` internally. This preserves backward compatibility — advanced users who already have git set up can still call `gcc-init.sh` directly.

## `setup.sh` Logic (pseudocode)

```
parse_args(target_dir, project_name, project_goal)
  → if missing: enter interactive wizard

if target_dir does not exist:
  mkdir -p target_dir

cd target_dir

if not inside a git repo:
  git init -b main
  git config user.email / user.name (if not set)

if git has no commits:
  git commit --allow-empty -m "initial"

copy scripts/ from GCC_SOURCE_DIR into target_dir/scripts/
make all scripts executable (chmod +x)

run: bash scripts/gcc-init.sh "$project_name" "$project_goal"

generate AGENTS.md  (agent-neutral GCC protocol)
generate CLAUDE.md
generate .cursor/rules
generate .windsurfrules
generate .github/copilot-instructions.md
generate .gemini/GEMINI.md
generate .agent/rules/gcc.md

print success summary
```

## `AGENTS.md` Content Changes

Current `CLAUDE.md` opening:
> *"You are operating with a structured, versioned context management system called GCC..."*

New `AGENTS.md` opening:
> *"This project uses GCC (Git Context Controller) — a structured, versioned memory system for AI agents..."*

All protocol rules (COMMIT, BRANCH, MERGE, CONTEXT, OTA logging) remain identical.

## Test Coverage

`tests/test_setup.bats` covers:
- Non-interactive mode with all args provided
- Interactive mode (piped input)
- Detects existing git repo, skips `git init`
- Detects missing initial commit, creates one
- All 7 agent files are created
- All agent files have identical content
- `scripts/` is copied and files are executable
- `.GCC/` is initialized
- Fails gracefully if GCC source dir not found
- Idempotency: second run warns but does not corrupt

---

## Out of Scope

- No package manager (npm/pip/brew) — pure Bash, zero dependencies
- No auto-update mechanism
- No Windows `.bat` wrapper (Git Bash is sufficient)
- No removal/uninstall script (not needed for MVP)
